# architecture.md 模板

**路径：** `docs/upstream/<project>/architecture.md`  
**模式：** 每次 research 全量覆盖

```markdown
# <project> 架构文档

> 最后更新：<ISO date> | Commit: <hash[:8] 或 n/a>

## 项目概述

**用途**：<一句话>

**核心特性**：
- <特性 1>
- <特性 2>

**适用场景**：<何时以及为何使用>

## 目录结构

\`\`\`
<精简目录树；忽略 node_modules/dist/build/.git>
\`\`\`

## 关键模块说明

### `<模块目录或文件>`
- **职责**：<该模块做什么>
- **关键导出/接口**：<主要 API 或类>
- **依赖关系**：<依赖哪些其他模块>

（按重要性列出 3–7 个核心模块）

## 核心概念

**<概念名>**：<解释>
（2–4 个项目独有的核心概念）

## 数据流 / 调用关系

<文字或简单 ASCII 描述主数据流或调用链>

## 与其他项目的关系

- **依赖**：<外部项目/库>
- **被依赖**：<典型使用场景>

## 给 LLM 的使用提示

使用此项目时需要注意：
1. <关键约束或陷阱>
2. <推荐的使用方式>
3. <常见错误>
```

面向 LLM 写作：解释「为什么」、点明约定、术语先定义；优先写模式，不要堆完整文件清单。
