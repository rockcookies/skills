# 生成架构文档

此技能为 `upstream/` 目录中的项目生成 LLM 友好的架构文档，帮助其他 Claude 实例（或 LLM）理解项目功能、组织结构和有效使用方式。

## 使用方式

此技能支持 `upstream/` 目录下的任意项目。执行时会自动：
1. 动态扫描 `upstream/` 目录获取可用项目列表
2. 根据用户指定的项目名称生成架构文档
3. 如未指定项目，列出所有可用项目供选择

## 工作流程

### 第一步：提取项目名称

从用户请求中识别要记录的上游项目名称，应与 `upstream/` 中的目录匹配。

如用户未指定项目，请其从可用选项中选择。

### 第二步：验证项目存在

1. 扫描 `upstream/` 目录获取所有可用项目
2. 检查 `upstream/{project-name}` 是否存在
3. 如不存在，列出所有可用项目（通过扫描结果）供用户选择

### 第三步：分析项目

收集项目信息：

1. **阅读 README**（如存在）：`upstream/{project-name}/README.md`
   - 提取项目目的和概述

2. **检查 CLAUDE.md**：`upstream/{project-name}/CLAUDE.md`
   - 提取项目特定指令和指南

3. **扫描目录结构**：使用 `ls -R` 或 `find` 获取完整树形结构
   - 识别关键目录：`skills/`、`agents/`、`commands/`、`hooks/`、`scripts/` 等

4. **读取关键配置文件**（如存在）：
   - `package.json` - 依赖和脚本
   - `tsconfig.json` - TypeScript 配置
   - 其他揭示项目结构的配置文件

### 第四步：生成架构文档

在 `docs/architecture/{project-name}-arch.md` 创建文件，使用此模板：

```markdown
# {项目名称} 项目架构

## 项目概述

**用途**：{一句话描述项目功能}

**核心特性**：
- {特性 1}
- {特性 2}
- {特性 3}

**使用场景**：{何时以及为何使用此项目}

## 项目结构

```
{仅显示关键目录和文件的简化目录树}
```

## 关键目录说明

### `{目录名}/`
- **用途**：{该目录的作用}
- **关键文件**：
  - `{文件}`：{作用}
  - `{文件}`：{作用}

### `{另一个目录}/`
- **用途**：{该目录的作用}

## 核心概念

**{概念 1}**：{解释}
- 项目中的作用：{重要性}

**{概念 2}**：{解释}
- 项目中的作用：{重要性}

## 与其他项目的关系

- **依赖**：{此项目依赖什么}
- **被依赖**：{什么依赖此项目}
- **相关项目**：{类似或互补的项目}

## 给 LLM 的提示

使用此项目时，LLM 应关注：
1. {关键方面 1}
2. {关键方面 2}
3. {关键方面 3}

## 文件参考

**项目位置**：`upstream/{project-name}/`

**相关文档**：
- {相关文档链接（如有）}
```

### 第五步：写入文档

使用 `Write` 工具在 `docs/architecture/{project-name}-arch.md` 创建架构文档。

### 第六步：确认并报告

告知用户：
- 文档创建成功
- 新文件的完整路径
- 所记录内容的简要摘要

## 示例会话

**用户**："为 obra-superpowers 生成架构文档"

**你的响应**：
1. 识别项目：`obra-superpowers`
2. 验证：`upstream/obra-superpowers/` 存在
3. 分析：
   - 读取 README.md → Superpowers 开发工作流
   - 扫描目录 → `skills/`、`agents/`、`commands/`、`hooks/`
   - 识别关键组件
4. 生成：`docs/architecture/obra-superpowers-arch.md`
5. 报告："已在 docs/architecture/obra-superpowers-arch.md 创建架构文档"

## 文档编写技巧

- **简洁而完整**：专注于 LLM 需要了解的内容，而非详尽细节
- **解释"为什么"**：不仅列出文件，还要解释其用途
- **使用通俗语言**：避免行话，除非必要，使用时加以解释
- **突出模式**：如项目遵循特定约定，请提及
- **面向 LLM**：构建信息结构，帮助其他 AI 快速理解项目目的和组织方式

## 错误处理

- 项目不存在：列出可用项目请用户选择
- README.md 不存在：仅使用目录分析继续
- 项目为空：在文档中注明，建议用户检查源代码
